# 🌿 Code Cultivation: Object-Oriented Garden Systems

This repository contains the development of **Module 01** of the 42 Python Piscina. This project shifts away from procedural execution to dive deep into **Object-Oriented Programming (OOP)**, using a botanical management system as a framework to understand modern software architecture.

### 🏛️ Technical Bridge: From C to Python
A key focus of this module has been the conceptual transition from low-level systems to OOP. Specific emphasis was placed on:

* **Encapsulation & Data Integrity:** While in **C**, encapsulation is manually achieved using **static structs** in `.c` files (hiding definitions in `.h` to create opaque types), in Python, we explored how classes and visibility conventions manage this security natively.
* **Attribute Management:** Moving from pointer manipulation of fixed structures to Python's properties and methods, allowing for a fluid yet rigorous control over the object's internal state.
* **Memory & Instantiation:** Understanding class instance creation versus a structure's `malloc`, recognizing the role of the Garbage Collector compared to manual memory management.
